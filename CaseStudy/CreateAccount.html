Effectiveness Creating Account on Dashboard

Facilitating a seamless account creation process on the dashboard to expedite B2B business approaches.

Role

Product Designer

Timeline

Q2 2023

Result

We achieved approximately 74.97% effectiveness with the new design for creating an account on our dashboard.

PROJECT BRIEF

We have a web dashboard for creating genetic test orders, analyzing the tests, and generating PDF reports of the final results. As is typical with dashboards, there are many account roles. We've received feedback that the user account creation process is overly detailed, so we need to revamp it to make account creation more user-friendly.

Here for the goals:
  • Simplify the account creation process and journey.
  • Remove redundant steps/fields.
  • The relevant pages

CHALLENGES/PROBLEM

Since the company will be venturing into B2B, where we will be selling SaaS, there is a need for improvement in the account creation aspect of our SaaS product. This includes the creation of accounts and the roles within them.

Here are objective points that i need to explore it:

Simplify the user journey

We want to create a simple user admin interface for account creation. However, the flow and user journey are still too long and not intuitive. Here are some issues I've encountered:

  • The user journey is too lengthy, making the account creation process ineffective.
  • There are two different pages that are related to each other.
Remove redundant steps/field

Effectiveness is achieved when we don't have to do the same thing twice. In this company's dashboard, we have found quite a few fields that could actually be optional or even removed, but they are set as required fields, preventing users from proceeding to the next step of account creation.

Update Relevant Pages

we can call it a lack of consistency among the entire pages when creating an account. There are pages that display pop-up fields while creating, and there are also pages in the form of standalone pages. This greatly confuses the users because they might think they are in a different application due to the lack of consistency.

Too many roles that we need to configure out

in our dashboard, we have some interconnected roles, so I need to figure out the role that we have and confirm what it is role doing in our pipeline. so for this project we focus on 4 roles:

  • Lab account: the role which is an entity without login and below this role there is lab staff
  • Lab staff account: the role that user to analyze the sample and check it before generate the sample to the PDF report
  • Provider account: the role under the labs but can create the order and see the overall workflow from physician, and under this role there is the physician account
  • Physician account: the role that can create the order and check the patient result that can verified the report from the lab, the physician is the first people who interacting with customers. so physician have a responsibility about the patient result

VALIDATION

We conduct Qualitative Research for this project, since this problem from internal user so it will easy to reach them and validate the problem. For the research we conduct at two time, first it’s User Research and the second its Usability Testing.

User Research

We conducted user research on 2 internal users who have admin dashboard accounts, and our goals for conducting user research are:

  • We want to know the users' habits when creating a new account.
  • We want to identify which fields they leave blank.
  • We want to identify any additional fields that may be needed, as admins sometimes make changes in the backend.

*Due to confidentiality issues, I am unable to provide the details in this case study.

DESIGN EXPLORATION

We have several stages in the design exploration and validation process to quickly gather feedback that aligns with user needs.

  • Phase 1 → Brainstorming User Research Result and Flow
  • Phase 2 → Iterative design after receiving feedback from another designer
  • Phase 3 → Iterative design based on User Testing results

Phase 1 → Brainstorming User Research Result and Flow

After obtaining the results of the User Research, I tried to explain the research findings and conduct brainstorming with other designers in the company. Here are the results of the brainstorming flow that was obtained:

Remove the Lab Staff Tab from the Sidebar and relocate the Lab Staff page's functions into the Lab section.

Labs Staff tab remove

The purpose is:
  1. Allowing users to create a Lab Staff account without the need to search for the lab each time.
  2. Simplifying the flow.
  3. Minimizing the number of tabs in the sidebar.

Remove the Physician Tab from the Sidebar and relocate the Lab Staff page's functions into the Lab section.

Physician tab remove

The purpose is:
  1. Allowing users to create a Lab Staff account without the need to search for the Provider each time.
  2. Simplifying the flow (similar to the Lab Staff tab).
  3. Minimizing the number of tabs in the sidebar.

Phase 2 → Iterative design after receiving feedback from another designer

We already have a system design when designing this, so the design process in phase 1 is faster in order to obtain feedback quickly. Here are the design results:

Labs Account

Labs screen before and after

Before:
  1. The use of pop-ups is inconsistent because some pages use pop-ups while others do not.
  2. Users do not understand what the locality field is, so they rarely fill it out.
  3. State and city sometimes take time to find, and the fields are in dropdown format, making it difficult for users to find them one by one.
After:
  1. Add a header and breadcrumb to make it clear to the user which page they are currently on.
  2. Create a consistent form field across all pages, so the "Create Account" page will be within the page itself and not as a pop-up.
  3. Improve the profile picture upload feature in the lab, which will be used to display in the report.

Labs Staff Account

Labs Staff screen before and after

Before:
  1. Inconsistent use of pop-ups because some pages use them and others don't.
  2. There are fields that are too detailed, such as birth date, gender, country, city, etc., which make it time-consuming for users to fill them out.
After:
  1. Add a header and breadcrumb to make it clear to the user which page they are currently on.
  2. Remove fields that were previously required to be filled in, such as gender, birth date, etc., based on user research findings and to improve the speed of user account creation.
  3. Add a "Add Another Account" button to allow users to create multiple accounts simultaneously. This is based on observations during user research that users may need to create several accounts at once.

Provider Account

Provider screen before and after

Before:
  1. The placement of the photo on the right side is quite different from other profiles, and users may overlook it because their focus is on the form.
  2. There are many fields that users do not fill out due to the lack of clarity in the fields. This statement is based on user research findings.
  3. The gender, birth date, and all personal information fields are rarely filled out by users, which slows down the account creation process.
After:
  1. Improvising photo profile upload
  2. Minimizing fields in the PIC section, only requiring name and email as identifiers for the owner/PIC of this provider account.

Physician Account

Physician screen before and after

Before:
  1. Users have to select a provider every time they want to create a new physician, which can be time-consuming for the users.
  2. The placement of the photo profile update layout is not appropriate, as users may overlook it due to their focus on the left form. Additionally, the photo is personal and can be removed by the admin user when creating an account.
  3. Physician qualifications are rarely filled in when users create an account, as this information may be personal and not required when creating an account by the admin user.
After:
  1. Consistency of form fields with other account creation forms.
  2. Changing MRC number to optional, as it is personal and physicians can fill it themselves.
  3. Adding a add another account button to facilitate users in creating multiple accounts simultaneously, as the admin may have a significant amount of physician data to input.

Other

Homepage screen before and after

Changes are made on the home page before the user creates an account, applicable to the creation of Lab, Lab staff, Provider, and Physician accounts. One example is in the Labs tab, here are the changes:

Before:
  1. There is a State column in the table, which is not necessary information as there is already a city column.
  2. The action column takes up too much space, so there may be a change in navigation to access lab details.
After:
  1. Modify the details navigation so that when the user clicks on the name of a lab, it will directly display the details of that lab.
  2. Add contact numbers for the labs, making it easier for users to access lab contacts in case of any issues.

Phase 3 → Iterative design based on User Testing results

We have iterated on the design again based on feedback. Here are the results of the improvisations:

Improve Lab create account page

Lab screen before and after

Before:
  1. Modifying the lab upload flow related to report types as shown in the after design.
After:
  1. Adding a Report type for the Lab so that the admin user can choose whether this lab is used internally or externally by the company. If it's for an external company, then they need to upload a photo/logo to be placed in the PDF report.

Improve Lab Staff create account page

Lab Staff screen before and after

Before:
  1. The width of the option field is not aligned with the upper field.
  2. Adding another button like the field, as if it cannot be clicked.
After:
  1. Aligning the width with other fields.
  2. Placing the "add another" button outside the frame to give the impression that each frame has a distinct form.

Improve Provider create account page

Provider screen before and after

Before:
  1. Uploading a photo is not a priority for filling out this create account provider form, so it doesn't need to be at the top.
After:
  1. Changing the Associated Lab field to a search-like interface, allowing users to easily find labs and choose multiple labs.
  2. Placing the upload logo below and adjusting its design to match the existing photo upload.
  3. Adding service activation based on user requests, as users typically make these changes in the backend.
  4. Separating the sections between Provider details and PIC Details for clearer and easier form filling.
  5. Changing Email to National ID for the PIC to enable tracking if there are issues with the provider account.

Improve Physician create account page

Physcian screen before and after

Before:
  1. Each country has different doctor identification requirements, making the MRC Number too specific for Singapore.
  2. Using a Switch button is not suitable for the intended context.
  3. The "add another account" button looks like a field.
After:
  1. Changing MRC Number to License Number for a more general approach.
  2. Switching from a Switch bar to a Checkbox and improving the copywriting to better fit the context.
  3. Placing the "add another account" button outside the form to make it more visible.

LESSON LEARNED

Break down large sections into small parts

When I worked on this project, I was quite concerned because the tasks were too extensive, and I was afraid of not achieving the desired goals.

However, breaking down this project into small parts, each with its own sub-goals, made it easier for me to tackle them one by one and ultimately achieve the goal of enhancing user effectiveness in creating accounts.

Don't forget the Edge Cases

In this project, we encountered quite a few edge cases that needed to be addressed. Understanding the project limitations and collaborating frequently with the project manager can help handle these edge cases either in the current iteration or plan for them in the next one.

It is crucial to collaborate with all relevant stakeholders and be proactive to ensure optimal results.
